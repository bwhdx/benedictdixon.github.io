# Mikaela Ikenouye · Cooking & Baking

Static, SEO-first food blog. Each recipe is authored as structured JSON +
prose HTML; the build pipeline renders both to static HTML pages with
`schema.org/Recipe` JSON-LD baked in.

See `PLAN.md` for architecture and decisions, `schema.md` for the recipe
schema, `TODO.md` for the work plan.

## Stack

- Static HTML + CSS, generated at build time via `react-dom/server`.
- **No client-side React.** Pure HTML in the browser, plus a tiny vanilla
  `client.js` for the metric/US toggle and persisted preference.
- Hosted on GitHub Pages (static), no CI, no server.

## Local dev

```bash
npm install
npm run build         # compile JSX, validate recipes, render static HTML
npm run photos        # generate responsive image variants from originals
npm run serve         # local static server on :8000
```

## Adding a recipe

```bash
# 1. Copy the template recipe directory (use an existing one as a model)
cp -r recipes/classic-chocolate-chip-cookies recipes/my-new-recipe

# 2. Edit recipe.json (see schema.md for required fields)
# 3. Edit essay.html (your thoughts on the dish; no <script>, no inline styles)
# 4. Replace hero.jpg with your shot

# 5. Generate sized photo variants and build
npm run photos
npm run build

# 6. Commit and push
git add recipes/my-new-recipe/
git commit -m "New recipe: my new recipe"
git push origin main
```

The build will fail if any recipe is missing required fields, or any
ingredient is missing its metric or US measurement. Fix and re-run.

## Layout

```

  Layout.jsx          HTML shell, head meta, site chrome
  RecipeComponents.jsx   Recipe primitives (hero, meta, ingredients, etc.)
  Pages.jsx           Home, RecipesIndex, About, RecipePage, NotFound
build/                Babel output (gitignored)
tools/
  build.cjs           Renders JSX → static HTML, generates RSS/sitemap/llms.txt
  validate.cjs        Recipe schema validation
  optimise-photos.cjs Sharp-based responsive image pipeline
recipes/
  <slug>/
    recipe.json       Structured data
    essay.html        Prose
    hero.jpg          Original photo
    index.html        Generated by build
site.css              Hand-written CSS (Phase 1 placeholder — replace in Phase 2)
client.js             Vanilla JS for unit toggle
index.html            Generated by build (home page)
recipes/index.html    Generated by build (recipes listing)
about/index.html      Generated by build
feed.xml              Generated by build (Atom)
sitemap.xml           Generated by build
llms.txt              Generated by build (LLM crawler index)
404.html              Generated by build
```

## SEO

Every page ships full static HTML — visible to Google, GPTBot, PerplexityBot,
Claude-Web, and any other crawler that doesn't execute JS. Per-recipe
`schema.org/Recipe` JSON-LD is rendered in `<head>` for rich-results
eligibility. `/llms.txt` provides a markdown index for LLM crawlers.
